# NumPy Project: E-Commerce Sales Analysis

## Project Overview

This project analyzes daily e-commerce order data using NumPy. The goal is to evaluate sales performance and generate meaningful business insights using statistical metrics.

## Problem Statement

An e-commerce company receives orders every day. Analyzing sales data manually is time-consuming and may lead to errors. This project provides a simple way to analyze daily sales data and monitor performance.

## Objectives

* Store daily order data using a 1D NumPy array.
* Calculate total and average orders.
* Identify highest and lowest sales days.
* Measure sales variation using sales range.
* Calculate median orders.
* Count days with above-average sales.

## Tools Used

* Python
* NumPy
* Jupyter Notebook

## Key Analysis

* Total Orders
* Average Orders
* Highest Orders
* Lowest Orders
* Sales Range
* Median Orders
* Above Average Sales Days

## Results

| Metric             | Value  |
| ------------------ | ------ |
| Total Orders       | 1110   |
| Average Orders     | 158.57 |
| Highest Orders     | 220    |
| Lowest Orders      | 110    |
| Sales Range        | 110    |
| Median Orders      | 150    |
| Above Average Days | 3      |

## Insights

* A total of 1,110 orders were received during the week.
* The average number of orders was around 159 per day.
* The minimum orders were 110, while the maximum orders were 220.
* Sales varied by 110 orders between the highest and lowest sales days.
* A typical day received around 150 orders based on the median value.
* Three days performed above the weekly average.

## Business Impact

* Helps monitor sales performance.
* Identifies peak and low sales periods.
* Supports inventory planning.
* Enables data-driven decision-making.

## Conclusion

This project demonstrates how NumPy can be used to analyze sales data efficiently and generate valuable business insights.
